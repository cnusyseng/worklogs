Installation of docker on Ubuntu VM


cat /etc/*release*

dpkg --print-architecture

* Removing old version of docker if they are already installed on the system

sudo apt remove $(dpkg --get-selections docker.io docker-compose docker-compose-v2 docker-doc podman-docker containerd runc | cut -f1)

# Add Docker's official GPG key:
sudo apt update
sudo apt install ca-certificates curl
sudo install -m 0755 -d /etc/apt/keyrings
sudo curl -fsSL https://download.docker.com/linux/ubuntu/gpg -o /etc/apt/keyrings/docker.asc
sudo chmod a+r /etc/apt/keyrings/docker.asc

# Add the repository to Apt sources:
sudo tee /etc/apt/sources.list.d/docker.sources <<EOF
Types: deb
URIs: https://download.docker.com/linux/ubuntu
Suites: $(. /etc/os-release && echo "${UBUNTU_CODENAME:-$VERSION_CODENAME}")
Components: stable
Architectures: $(dpkg --print-architecture)
Signed-By: /etc/apt/keyrings/docker.asc
EOF

sudo apt update

sudo apt install docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin

sudo systemctl status docker

sudo systemctl start docker

sudo docker run hello-world

Docker Init
--------------
Docker has a built in scaffolder that does most of the boilerplate for you. It's called Docker init.

Docker init does below things.
1) When do CD into the project and run Docker init and it walks you through a short interactive prompt. It looks at the files in your project, guesses what language you're using
2) Package.JSON means node
requirement.txt means python etc and ask you to confirm
3) What version of Node do you want to use ? 20
4) which package manager do you want to use ? npm
5) what command do you want to use to start the app? node src/index.js
6) what port does your server listen on ? 3000

you will get four files
1) Dockerfile
2) .dockerignore
3) compose.yml
4) README.Docker.md

vim Dockerfile
ARG NODE_VERSION=20
FROM node:${NODE_VERSION}-alpine AS base
WORKDIR /usr/src/app

FROM base AS deps
RUN --mount=type=cache, target=/root/.npm \
	npm ci --omit=dev

FROM deps AS final
ENV NODE_ENV=production
USER node
COPY . .
EXPOSE 3000
CMD ["node", "src/index.js"]

Using ENTRYPOINT with CMD
-------------------------

vim Dockerfile.sleeper-v2
FROM ubuntu
ENTRYPOINT ["sleep"]

This is argument where you can pass sleep duration at the execution or starting of the image

$ docker run ubuntu-sleeper 10

   sleep       10
<ENTRYPOINT>  <CMD>

ENTERYPOINT + CMD Together, Then Override
-------------------------------------------
$ vim Dockerfile.sleeper-v3
FROM ubuntu
ENTERYPOINT ["sleep"]
CMD ["5"]

Here if nothing is passed as an argument will use 5 as a argument, if argument is passed it will be used

$ docker run ubuntu-sleeper 10
# here the sleep time will be 10 sec

    sleep      10
 <ENTRYPOINT> <CMD>

# docker run ubuntu-sleeper
# here the sleep time will be 5 sec which uses CMD 5
 
   sleep        5
 <ENTRYPOINT>  <CMD>


Modify ENTRYPOINT at run time

$ docker run --entrypoint sleep2.0 ubuntu-sleeper 10
This command will over ride

   sleep2,0    10
 <ENTRYPOINT> <CMD>

Attach it to the clicknet Docker network that you created in the previous task.

$ docker run -d --network=clicknet --name redis redis:alpine
$ docker ps 

Next, create a simple container named clickcounter using the image kodekloud/click-counter, attach it to the clickcounter network from the earlier task, and then expose it on host port 8085.
The clickcounter app runs on port 5000.
$ docker run -d -p 8085:5000 --network clicknet --name clickcounter kodekloud/click-counter
$ docker ps

Let's clean up the resources created in the previous steps.
Delete the redis and clickcounter containers, alongwith the clicknet network you had created.
$ docker ps
$ docker stop ff847
$ docker stop a2530
$ docker ps -a
$ docker rm a2530b301ef5
$ docker rm ff847b0ec53e
$ docker ps -a
$ docker network rm clicknet


Create a compose.yaml file in the /root/clickcounter directory. Once done, bring up the stack using Docker Compose.
The compose file should have the following exact specification:
redis service - Use redis:alpine image name
clickcounter service - Use the kodekloud/click-counter image name; the app runs on port 5000 and should be exposed on host port 8085

cat compose.yaml
services:
  redis:
    image: redis:alpine
  clickcounter:
    image: kodekloud/click-counter
    ports:
      - 8085:5000

$ docker compose -f compose.yaml up
