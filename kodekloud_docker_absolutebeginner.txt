Installation of docker on Ubuntu VM


cat /etc/*release*

dpkg --print-architecture

* Removing old version of docker if they are already installed on the system

sudo apt remove $(dpkg --get-selections docker.io docker-compose docker-compose-v2 docker-doc podman-docker containerd runc | cut -f1)

# Add Docker's official GPG key:
sudo apt update
sudo apt install ca-certificates curl
sudo install -m 0755 -d /etc/apt/keyrings
sudo curl -fsSL https://download.docker.com/linux/ubuntu/gpg -o /etc/apt/keyrings/docker.asc
sudo chmod a+r /etc/apt/keyrings/docker.asc

# Add the repository to Apt sources:
sudo tee /etc/apt/sources.list.d/docker.sources <<EOF
Types: deb
URIs: https://download.docker.com/linux/ubuntu
Suites: $(. /etc/os-release && echo "${UBUNTU_CODENAME:-$VERSION_CODENAME}")
Components: stable
Architectures: $(dpkg --print-architecture)
Signed-By: /etc/apt/keyrings/docker.asc
EOF

sudo apt update

sudo apt install docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin

sudo systemctl status docker

sudo systemctl start docker

sudo docker run hello-world

Docker Init
--------------
Docker has a built in scaffolder that does most of the boilerplate for you. It's called Docker init.

Docker init does below things.
1) When do CD into the project and run Docker init and it walks you through a short interactive prompt. It looks at the files in your project, guesses what language you're using
2) Package.JSON means node
requirement.txt means python etc and ask you to confirm
3) What version of Node do you want to use ? 20
4) which package manager do you want to use ? npm
5) what command do you want to use to start the app? node src/index.js
6) what port does your server listen on ? 3000

you will get four files
1) Dockerfile
2) .dockerignore
3) compose.yml
4) README.Docker.md

vim Dockerfile
ARG NODE_VERSION=20
FROM node:${NODE_VERSION}-alpine AS base
WORKDIR /usr/src/app

FROM base AS deps
RUN --mount=type=cache, target=/root/.npm \
	npm ci --omit=dev

FROM deps AS final
ENV NODE_ENV=production
USER node
COPY . .
EXPOSE 3000
CMD ["node", "src/index.js"]

Using ENTRYPOINT with CMD
-------------------------

vim Dockerfile.sleeper-v2
FROM ubuntu
ENTRYPOINT ["sleep"]

This is argument where you can pass sleep duration at the execution or starting of the image

$ docker run ubuntu-sleeper 10

   sleep       10
<ENTRYPOINT>  <CMD>

ENTERYPOINT + CMD Together, Then Override
-------------------------------------------
$ vim Dockerfile.sleeper-v3
FROM ubuntu
ENTERYPOINT ["sleep"]
CMD ["5"]

Here if nothing is passed as an argument will use 5 as a argument, if argument is passed it will be used

$ docker run ubuntu-sleeper 10
# here the sleep time will be 10 sec

    sleep      10
 <ENTRYPOINT> <CMD>

# docker run ubuntu-sleeper
# here the sleep time will be 5 sec which uses CMD 5
 
   sleep        5
 <ENTRYPOINT>  <CMD>


Modify ENTRYPOINT at run time

$ docker run --entrypoint sleep2.0 ubuntu-sleeper 10
This command will over ride

   sleep2,0    10
 <ENTRYPOINT> <CMD>

Lab: Docker Registry

What is a Docker Registry?
A Service that store and distributes container images

By default, the Docker engine interacts with ?
DockerHub

DockerHub is a hosted registry solution by Docker Inc.
Besides public and private repositories, it also provides:
    * Automated image builds
    * Integration with source control platforms like:
        * GitHub
        * Bitbucket
    * Webhooks
    * Image versioning and tagging
    * Team collaboration and access control
    * Vulnerability scanning (on some plans)

Which command is used for login to a self-hosted registry?
docker login [SERVER]

Let's practice deploying a registry server on our own.
Run a registry server with name equals to my-registry using registry:2 image with host port set to 5000, and restart policy set to always.
Note: Registry server is exposed on port 5000 in the image.
Here we are hosting our own registry using the open source Docker Registry.
NOTE: You do not need to run docker login for this local registry setup because authentication is not configured. This registry allows for anonymous access.
$ docker run -d -p 5000:5000 --restart=always --name=my-registry registry:2

Now, it's time to push some images to our registry server. Let's push two images for now: nginx:latest and httpd:latest.
Note: Don't forget to pull them first.

$ docker image pull nginx:latest
$ docker image tag nginx:latest localhost:5000/nginx:latest
$ docker image push localhost:5000/nginx:latest
$ docker image pull httpd:latest
$ docker image tag httpd:latest localhost:5000/httpd:latest
$ docker image push localhost:5000/httpd:latest

To check the list of images pushed, use the following command:
$ curl localhost:5000/v2/_catalog

Let's remove all the dangling images we have locally.
Use prune to remove them.
How many images do we have now?
Note: Make sure we don't have any running containers except our registry-sever.
$ docker container prune


Now, we can pull images from our registry-server as well. Use docker pull [server-addr/image-name] to pull the images that we pushed earlier.
In our case, we can use:
$ docker pull localhost:5000/nginx

Finally, let's clean up our registry.
Stop and remove the my-registry container.
$ docker stop 3c4eaa
$ docker rm 3c4eaa
